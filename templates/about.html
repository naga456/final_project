About my project:


I really wanted to challenge myself and try to solve different integration problems. I thought the job searching process was very frustrating and I wanted to see if there was a better way of doing it using what we learned about data analytics and machine learning. This application takes a resume or description about your work experience and compares it using probability methods to classify job categories. Job descriptions were pulled for 5 different job categories and used to train a naive bayes model.

Design:

The Visualization:

The website takes in a text input and figure outs the important words. Once we classify and predict your job category, we compare your key words to job descriptions within your job category. The size of the word displayed is using the TFIC weight and is used to visualize the word's strength in relation to its importance in prediction.

Usage:

  1. Load the data into pandas and clean it up.
  2. Save the data to csv.
  3. Train the model against the clean data.
  4. Save the model to be used later.
  5. Build the database in PGAdmin.
  6. Load the clean data csv.
  7. Load the webpage and the user can copy their resume into the text area.
  8. The HTML passes the input to Flask.
  9. Clean the input using the same data preprocessing used to train the model.
  10. Use the prediction to query the database for a list of job descriptions matching the prediction.
  11. Apply TFIDF (term frequency-inverse document frequency) to the list to find the word weights.
  12. Create a dictionary of words and weights.
  13. Flask sends the prediciton and dictionary to the HTML page.
  14. HTML page displays the prediciton.
  15. HTML page sends the dictionary to JavaScript to make the visualization.
  16. The visualization is displayed on the HTML page.

Interesting Findings:

  1. Without stopwords, the model had a score of 65%.
    With stopwords cleaning the model had a score of 67%
  2. I'm not 100% sure the word weights are the weights to calculate the model.
  3. Text analysis and classification didn't require a large dataset (as low as 2000 documents).
