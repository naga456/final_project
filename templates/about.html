About this application

About my project: I really wanted to challenge myself and take on integration problems head on. I thought the job searching process was very frustrating and I wanted to see if there was a better way of doing it using what we learned about data analytics and machine learning. This application takes a resume or description about your work experience and compares it using concepts similar to sentiment analysis to classify job categories. Job descriptions were pulled for 5 different job categories and used to train a naive bayes model. The visualization: Takes in the text input and figure outs the important words. Once we classify and predict your job category, we compare your key words to job descriptions within your job category. The size of the word is used to find its strength in relation to using the word to predict using TFID weight. Add api and webscraping for LinkedIn and Indeed real time job description scraping. Apply additional model tuning such as stemming, ngrams, add additional categories Add additional datasets Error handling Interesting Findings: 1. Without stopwords, the model had a score of 65%. Text analysis didn't require that large of a dataset, nor did each classification. Usage: 01. load the data into pandas to clean it up. 02. save the data to csv. 03. train the model against the clean data. 04. save the model to be used later. 05. build the database in PGadmin 06. Load the clean data csv. 07. Load the webpage and the user can copy their resume into the text area. 08. The HTML passes the input to Flask 09. Clean the input using the same method as the model. 10. Use the prediction to query the database for a list of job descriptions matching the prediction. 11. Apply TFID to the list to find the word weights. 12. Create a dictionary of words and weights. 13. Flask sends the prediciton and dictionary to the HTML page. 14. HTML page displays the prediciton. 15. HTML page sends the dictionary to JavaScript to make the visualization. 16. The visualization is displayed on the HTML.
